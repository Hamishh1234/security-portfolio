# Evidence Summary – B20

For this activity, I reviewed how the security of my GitHub project could be improved.

The two main improvements I identified were adding a SECURITY.md file and adding or updating a .gitignore file.

## 1. SECURITY.md file

A SECURITY.md file would explain how security issues should be reported.

It would include things like:
- security issues should not be posted publicly
- issues should be reported privately
- private information such as passwords, API keys, private keys or account details should not be uploaded

## 2. .gitignore file

A .gitignore file helps stop unnecessary or sensitive files from being uploaded to GitHub.

It could include:
- .DS_Store
- .env
- log files
- node_modules
- Python cache folders

## What this would improve

These changes would make the project safer by reducing the chance of private files being uploaded and by giving a clear process for reporting security issues.
