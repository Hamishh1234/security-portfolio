# Activity B20 – Enhance GitHub Project Security

## Objective
To enhance the security of a GitHub project

## What I did
I reviewed how I could improve the security of my GitHub project and wrote a plan for changes that could be applied.

I did not fully apply the changes before submission, but I identified what should be added to make the repository safer.

## Findings
One improvement would be adding a SECURITY.md file. This file would explain how security issues should be reported. This is useful because security problems should not always be posted publicly, especially if they could expose private information.

Another improvement would be adding or updating a .gitignore file. This helps stop unnecessary or sensitive files from being uploaded to GitHub.

The .gitignore file should include things like .DS_Store files, environment files, log files, node_modules folders and Python cache folders.

These changes would improve the project because they reduce the chance of private files being uploaded and make the repository safer to manage.

## Evidence collected
I included a written security improvement plan in the evidence folder.

## Reflection
This activity made me realise that GitHub security is not only about the code itself. Even small files like SECURITY.md and .gitignore can help make a project safer and more organised.
