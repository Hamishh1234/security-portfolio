# Activity B11 – Cybersecurity Expert Discussion

## Objective
To talk to cybersecurity or technology industry people and find out about their current work or projects

## What I did
I spoke to one person with technology industry experience.

The person I spoke to is my friend who has completed a Master’s in Software Engineering and now works in the industry.

## Findings
He explained a personal project he was working on for his job. The project was designed to help sort information faster, instead of him having to manually sort through it himself.

Even though the project was not mainly a cybersecurity project, we talked about how security still matters when building software tools for work.

He explained that if software is handling work-related information, then the developer still has to think about how the data is accessed, stored and processed.

We also talked about how automation can save time, but it still needs to be built carefully. If a tool stores information badly, gives access to the wrong person, or processes information incorrectly, it could create problems.

This made me realise that cybersecurity is not only used in obvious cybersecurity jobs. It also connects to normal software engineering work, especially when projects handle data or automate work processes.

## Reflection
This activity helped me see how cybersecurity connects to software engineering. Even when someone is building a tool to make work faster, they still need to think about data handling, access control and whether the system is safe to use.
