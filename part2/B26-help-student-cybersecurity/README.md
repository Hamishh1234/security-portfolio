# Activity B26 – Helping Another Student

## Objective
To help another student in this unit understand a cybersecurity concept

## What I did
I helped another student in my unit understand website form spamming.

## Findings
I explained that website form spamming is when bots or attackers repeatedly submit forms on a website.

This can happen through contact forms, sign-up forms, login forms, or comment sections.

I explained that this can create problems such as fake accounts, spam messages, database clutter, and overloaded inboxes.

I also explained that form spam can become a security issue if the website does not properly validate or limit submissions.

We talked about ways to reduce the risk, such as CAPTCHA, rate limiting, input validation, email verification, honeypot fields, and blocking repeated submissions.

## Evidence collected
I included a short written summary of what I explained in the evidence folder.

## Reflection
This activity showed me that even simple website features like forms can become a security issue if they are abused. Explaining it to someone else helped me understand it better as well, because I had to explain both the problem and the protections in a simple way.
