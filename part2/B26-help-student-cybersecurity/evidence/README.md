# Evidence Summary – B26

For this activity, I helped another student in my course understand the cybersecurity concept of website form spamming.

This came up while working near our project, but the activity I am counting here is the explanation of the security concept, not the project work itself.

I explained that website form spamming is when bots or attackers repeatedly submit forms on a website. This could include contact forms, sign-up forms, login forms, or comment sections.

I explained that form spam can cause problems such as fake accounts, spam messages, overloaded inboxes, database clutter, and sometimes even security risks if the form does not properly validate input.

I also explained some ways websites can reduce form spam, such as using CAPTCHA, rate limiting, input validation, email verification, hidden honeypot fields, and blocking repeated submissions.

The main point I explained was that forms are useful for users, but they can also be abused if there are no protections in place.
