# Activity B3 – Proactive Security Implementations

## Objective
To discover 3 proactive security implementations in practice

## What I did
I looked for examples of security measures that are used before an attack or issue happens. I tried to choose examples that help prevent problems instead of only fixing them after.

## Findings
One proactive security implementation is automatic backups. Backups save copies of important files so they can be recovered if something goes wrong. This is useful if a device is lost, damaged, or affected by something like ransomware.

Another example is security awareness training. This teaches people about risks like phishing emails, scam messages and weak passwords before they become a problem. This is proactive because it can stop people making mistakes that attackers rely on.

Login alerts are also proactive. If an account sends an alert about a new or suspicious login, the user can notice quickly and take action. This can help stop unauthorised access before more damage is done.

## Evidence collected
I included screenshots showing examples of proactive security measures in the evidence folder.

## Reflection
This activity showed me that proactive security is about being prepared before something bad happens. Things like backups, training and login alerts can help reduce damage or stop attacks earlier.
