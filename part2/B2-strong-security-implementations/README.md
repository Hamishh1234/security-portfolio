# Activity B2 – Strong Security Implementations

## Objective
To discover 5 strong security implementations

## What I did
I looked at different security features that are used in apps, websites and accounts. I tried to choose examples that were different from the weak security examples I used in B1.

## Findings
One strong security implementation is end-to-end encryption. This is used in messaging apps and it means that only the sender and receiver should be able to read the messages. This helps protect private conversations.

Another example is passkeys. Passkeys are used as a safer way to sign into accounts because they do not rely on remembering or reusing a normal password. They can use things like fingerprint, face unlock or a device security key.

App permissions are also a strong security feature. Apps need permission before they can access things like the camera, microphone, location or contacts. This helps stop apps from accessing private information without the user knowing.

CommBank transaction alerts are another example. These alerts can warn the user about account activity or spending. This helps the user notice unusual activity faster.

GitHub security alerts are also a strong implementation. They can warn users if a project has a vulnerable dependency or another security issue that needs to be fixed.

## Evidence collected
I included screenshots showing examples of these strong security implementations in the evidence folder.

## Reflection
This activity showed me that strong security features are used in a lot of normal apps and websites. A lot of them work in the background, but they still help protect users from risks like stolen passwords, suspicious account activity or apps accessing too much information.
