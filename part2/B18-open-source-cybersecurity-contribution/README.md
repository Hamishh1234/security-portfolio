# Activity B18 – Open Source Cybersecurity Contribution

## Objective
To contribute to an open-source project related to cybersecurity

## What I did
I reviewed an open-source cybersecurity project on GitHub and wrote a proposed documentation improvement.

The project I looked at was Cyber-Drill-Tool, which is a cybersecurity awareness and simulation toolkit.

I did not submit the change as a pull request or issue before submission, but I wrote what improvement I would suggest for the project.

## Findings
The project includes cybersecurity learning tools such as phishing simulations, password checking, port scanning, XSS demos and SQL injection testing.

Because these tools are related to cybersecurity testing and simulations, I thought the documentation could be improved by adding clearer safety and ethical use notes for beginners.

The suggested improvement was to add a section explaining that the tools should only be used for education, controlled labs, or systems the user owns or has permission to test.

This would help beginners understand that cybersecurity tools should be used responsibly and not against real targets without permission.

## Proposed contribution
My proposed documentation improvement was:

Add a clearer beginner-friendly safety note near the tool descriptions.

The note could include:
- only use the tools for education or authorised testing
- do not run scans or simulations against real targets without permission
- phishing demos should use fake data only
- the project is for learning defensive security concepts

## Evidence collected
I included the GitHub project link and this proposed documentation improvement.

## Reflection
This activity showed me that open-source contributions do not always have to be code. Documentation is also important, especially in cybersecurity projects where beginners need to understand safe and ethical use. Even though I did not submit the contribution, reviewing the project helped me think about how cybersecurity tools should be explained responsibly.
