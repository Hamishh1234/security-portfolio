# Activity B22 – Enhance Community Website Security

## Objective
To enhance the cybersecurity of a website from my community

## What I did
I looked at a local community/business website and wrote security recommendations that could improve the login system.

## Findings
I focused on the Perth DJ Hub website because it is a local website that I already use.

When looking at the login page, I did not see any visible CAPTCHA or extra protection on the sign in screen.

This could be a risk because login pages can be targeted by repeated login attempts if there are not enough protections in place.

To enhance the website security, I wrote a short list of recommended improvements. These included adding CAPTCHA after several failed attempts, adding login delays or temporary lockouts, sending warning emails after failed logins, and adding two-factor authentication.

These changes would help reduce the risk of brute force login attempts and improve account protection.

## Evidence collected
I included a screenshot of the login page and a written security recommendation file in the evidence folder.

## Reflection
This activity showed me that improving cybersecurity does not always mean fully rebuilding a website. Even small changes to login security can make accounts safer and reduce common risks.
