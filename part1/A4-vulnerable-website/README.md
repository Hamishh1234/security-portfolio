# Activity A4 – Vulnerable Website

## Objective
To identify a website with a potential security issue

## What I did
I explored a website and looked at how the login system behaves when entering incorrect passwords

## Findings
The website I decided to try was Perth DJ Hub. This is a small business in Perth, meaning it may have weaker security then big businesses. Thats why i chose it.

When I entered incorrect passwords multiple times, I didn’t see any CAPTCHA or anything stopping me from trying again. It just kept letting me attempt to log in.

I also noticed that when using my own account email (as i have my own account with them) I didn’t get any warning emails or notifications after multiple failed login attempts.

Because of this it seems like there isn’t much visible protection against repeated login attempts. This could be a risk because someone could keep guessing passwords over time (Brute Force attack).

There could be protections running in the background that aren’t visible to me. But from what i can see this seems like a vulnerability.

After some research i learnt that such small vulnerabilities can make websites unsecure. This is a website i have been signed up to for a long time and i never thought it would be unsecure.
